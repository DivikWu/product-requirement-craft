# Problem Framing — Yami 英文站首页优化

## Problem Statement

对 Yami 不够熟悉的用户，包括新访客、低频回访者和初期购买者（WHO），
在进入 Yami 英文站首页时（WHEN），
面临首页未能优先、快速建立清晰的平台范围认知，且前部内容以促销和商品货架为主、导航入口混合了品类/场景/热搜等多种维度、信任信号出现较后等问题（WHAT Problem）。
他们希望通过首页快速判断平台卖什么、覆盖哪些品类、是否适合自己、能否放心购买，并找到合适的浏览或购买起点（WHAT Job）。
我们的方案需要帮助用户更快理解平台定位与价值，更容易建立信任，更顺畅地开始浏览或购买（WHAT benefits for the customer），
同时提升首页新客转化率（WHAT benefits for the company）。

---

| Field | Description |
|-------|-------------|
| **WHO** | 对 Yami 不够熟悉的用户，包括新访客、低频回访者和初期购买者 |
| **WHAT Problem** | 首页未能优先、快速建立清晰的平台范围认知；前部内容以促销和商品货架为主；导航入口混合了品类/场景/热搜等多种维度；信任信号出现较后 |
| **WHEN** | 进入 Yami 英文站首页时 |
| **WHAT Job** | 快速判断平台卖什么、覆盖哪些品类、是否适合自己、能否放心购买，并找到合适的浏览或购买起点 |
| **WHAT benefits for the customer** | 更快理解平台定位与价值，更容易建立信任，更顺畅地开始浏览或购买 |
| **WHAT benefits for the company** | 提升首页新客转化率 |

## Why this is a good example

- **WHO** is specific — not "all users" but a defined segment with clear reasoning
- **WHAT Problem** lists multiple concrete, observable issues — not vague "bad experience"
- **WHEN** pinpoints the exact touchpoint — "entering the English homepage"
- **WHAT Job** uses JTBD language — what the user wants to achieve, not what the product should do
- **Customer Benefit** is experiential — describes how the user's experience improves
- **Company Benefit** is measurable — points to a specific metric (conversion rate)
- **Problem Statement** reads as a fluent paragraph, not a mechanical fill-in-the-blank
